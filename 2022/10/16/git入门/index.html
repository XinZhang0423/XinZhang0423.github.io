1. git环境配置：

国内需要换源进行镜像下载不然下载速度慢这里略过。下载git，或者使用vscode内置git。使用方法就是在命令行中输入指令。

2.git基本原理（核心）：

![截屏2022-09-16 12.51.18](/Users/zhangxin/Library/Application Support/typora-user-images/截屏2022-09-16 12.51.18.png)

git主要分为四个部分：directory=目录，repository=存储库

  1. 工作目录：working directory，工作区本地存放代码的地方，在工作目录运行git会出现 .git隐藏文件夹，里面的是暂存区，和一些文件目录。
  2. 暂存区: stage/index，本质上只是一些文件，保存文件列表信息
  3. 资源库: git directory，repository
  4. 远程工作目录remote directory

master主分支

基本操作：

![截屏2022-09-16 13.04.33](/Users/zhangxin/Library/Application Support/typora-user-images/截屏2022-09-16 13.04.33.png)

（1）新建仓库：

git init:在当前目录下创建.git文件，初始化一个git repository

git clone 网址url：在当前目录下，克隆一个url上repository，注意复制的时候，不要复制到换行，不然直接执行了，执行后在当前目录下下载了网址上的文件。

（2）文件操作：

git作为版本控制系统，本质上就是对文件的管理和操作，文件的状态包括：

  1. 未跟踪（untracked）：此文件在本地文件夹，但从未加入过stage，未参与版本控制系统，需要：git add filename把这个文件添加到缓存区，其实就是记录下这个文件的地址，改变文件的状态为staged，，这样就被跟踪了可以git commit进git仓库。

  2. 已修改（unmodified）：此文件已入库，且入库文件和当前文件是一样的，可以被修改（modified），或者git rm移出库变成untracked。

  3. 已缓存(modified)：此文件已入库且被修改在本地产生了新版本，可以通过git add变成staged模式。git checkout状态回滚。

  4. 已提交(commited):git commit

分支是用来管理平行的多个版本的。例如在主分支master外创建新的分支用来存放新的版本内容。
